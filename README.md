# Система Образовательный портал учреждения

Универсальное веб-приложение для управления образовательными программами, подходит для школ, вузов, корпоративного обучения и тренингов.

## Возможности

### Для организаторов
- Создание и управление программами обучения
- Добавление и управление участниками
- Создание задач различных категорий
- Оценка работ участников
- Предоставление обратной связи и отзывов
- Отслеживание прогресса участников

### Для участников
- Просмотр программ и задач
- Отправка ответов на задачи с файлами
- Просмотр своих оценок
- Получение обратной связи от организаторов
- Просмотр предстоящих дедлайнов
- Отслеживание своего прогресса

## Используемые технологии

- **Бэкенд**: Python, Flask
- **База данных**: PostgreSQL
- **Фронтенд**: HTML5, CSS3, Bootstrap 5
- **Аутентификация**: Flask-Login
- **ORM**: Flask-SQLAlchemy
- **Деплой**: Docker, Docker Compose

## Установка и настройка

### Требования
- Docker
- Docker Compose

### Быстрый старт

1. Клонируйте репозиторий:
```bash
git clone https://github.com/yourusername/electronic-diary.git
cd electronic-diary
```

2. Соберите и запустите контейнеры:
```bash
docker compose up --build
```

3. Откройте приложение по адресу:
```
http://localhost:5000
```

### Ручная установка (без Docker)

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Настройте переменные окружения (создайте файл `.env`):
```
DATABASE_URL=postgresql://diary_user:diary_password@localhost:5432/diarydb
SECRET_KEY=your-secret-key-change-in-production
```

3. Запустите приложение:
```bash
python app.py
```

## Схема базы данных

Приложение использует следующие модели базы данных:

- **User**: Хранит информацию о пользователях (организаторы и участники)
- **Program**: Представляет программы обучения
- **Enrollment**: Связь "многие-ко-многим" между участниками и программами
- **Task**: Задачи, назначаемые программам
- **TaskSubmission**: Отправки ответов участников на задачи
- **TaskFeedback**: Отзывы организаторов на работы участников
- **Grade**: Оценки участников за задачи
- **Notification**: Уведомления для пользователей

## Конфигурация

### Переменные окружения

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `DATABASE_URL` | URL подключения PostgreSQL | `postgresql://diary_user:diary_password@db:5432/diarydb` |
| `SECRET_KEY` | Секретный ключ Flask для сессий | `your-secret-key-change-in-production` |
| `FLASK_APP` | Точка входа Flask приложения | `app.py` |

### Безопасность

- Измените `SECRET_KEY` в продакшене
- Используйте надежные пароли для учетных данных базы данных
- Рассмотрите возможность использования HTTPS в продакшене

## Деплой

### Продакшн с Docker

```bash

docker compose up --build -d

# Остановка контейнеров
docker compose down
```

2. Использования production WSGI сервера
3. Добавления обратного прокси (Nginx)
